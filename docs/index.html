The problem

Cooling is a control problem

Data centers spend a huge share of their power just on cooling. Too little and servers overheat; too much and you waste energy. It's a continuous balancing act — a perfect fit for reinforcement learning.

An environment

A custom Gymnasium env models room temperature with a heat-balance equation: IT load heats it up, cooling pulls it down, ambient air leaks in.

An agent

A Soft Actor-Critic policy learns from scratch by trial and error — no hand-coded rules — discovering when and how hard to run the cooling.

A reward

Every minute, the agent is scored on how close it held the target, how much power it used, and whether it crossed the thermal safety line.

Architecture

The control loop

Each 60-second tick, the agent picks a cooling effort, the simulated room reacts, and a reward flows back. Repeat 1,440 times for one simulated day.

The physics

Heat balance

Room temperature evolves with a simple first-order heat equation. Servers add heat, the agent's cooling removes it, and the warmer outside air constantly leaks in.

ΔT = k ·( heatIT − coolaction ) + leakambient
heatIT rises with server load (0–1) · coolaction rises with the agent's effort (0–1) · leak is a slow drift toward the 28 °C outside · k scales how fast the room responds.
The objective

Reward function

Each step returns a negative cost: squared distance from target, a small energy term, and a heavy penalty for crossing the 30 °C safety ceiling.

r = −( (T − 24)² + 0.02·power )
if T > 30 °C → r −= 10
Because being even 1 °C off costs 1.0 while max energy costs only 0.4, the agent prioritizes holding temperature, then economizes on power once it's close.

Get started

Run it locally

Pure Python — clone, install, and you can sanity-check the sim, train an agent, and watch it work in three commands.

01 / SANITY

Test the environment

Runs the sim with random actions and live plots — no RL needed. Confirms everything works.

02 / TRAIN

Train the SAC agent

Normalizes observations, trains for 300K steps, then saves the policy and normalization stats.

03 / EVALUATE

Watch it cool

Loads the trained model, runs one full day deterministically, and renders the result to a plot.

# 1 — install dependencies
pip install -r requirements.txt
cd environment

# 2 — sanity-check the simulation (random actions)
python test_env_run.py

# 3 — train the Soft Actor-Critic agent (writes sac_dc.zip + vecnorm.pkl)
python train_sac.py

# 4 — load the policy and render one episode → eval_plot.png
python eval_and_render.py
